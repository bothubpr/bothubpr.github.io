【系列连载番外篇】打通最后一公里：从本地极客网盘到 GitHub Pages 的“全自动发车”

哈喽，各位极客小伙伴们！这是咱们自动化博客系列的 【特别番外篇】！ 在前四篇连载中，俺们在本地局域网里搭建了一个完美的“全自动知识库”。但独乐乐不如众乐乐，作为极客，咱们的终极目标肯定是把这套硬核架构 【推向公网】。 今天，俺就来扒一扒，如何白嫖 GitHub 的云端算力，并给咱们的 Python 门卫装上“火箭推进器”，实现真正的：网盘拖拽即发车，全球 CDN 秒级同步！ ★ 痛点破局：静态博客的“时空锁”与繁琐的 Push 只要你玩过 Hugo 或 Hexo 这类静态博客，你一定懂这种痛苦： 每次写完文章，不仅要手动敲 hugo 命令生成静态页面（HTML），还要手动 git add .、git commit、git push 把它们推送到 GitHub 上。 更要命的是 【时空折叠悖论】：如果你写了一篇日期标为明天的文章，本地不加特殊参数，它根本就不会生成！这意味着，你每天还得定个闹钟，按时去敲命令发布。这简直是 【反人类的伪自动化】！ ★ 降维打击第一步：云端流水线 (GitHub Actions) 换家 为了彻底解决每天手动打包和“未来文章”无法定时发布的问题，俺决定进行 【架构大换血】： 以后，俺们再也不在本地生成静态页面了，俺们要把原稿直接推上云端，让 GitHub 的超级计算机替咱们干活！ 在你的博客根目录下新建 .github/workflows/hugo.yaml，注入这套 CI/CD 黑魔法： name: Deploy Hugo site to Pages on: push: branches: ["main"] schedule: - cron: '30 22 * * *' # 每天北京时间早上 6 点半自动运行一次！ # ... (省略部分权限配置) jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 with: submodules: true fetch-depth: 0 - name: Setup Hugo uses: peaceiris/actions-hugo@v3 with: hugo-version: 'latest' extended: true - name: Build # 【核心魔法】：强行加入 --buildFuture，每天定时打破时空锁！ run: hugo --minify --buildFuture # ... (省略部署步骤) 有了它，哪怕俺这几天全在睡大觉，只要定时的 Cron 任务一触发，未来日期的文章就会全自动“破壳而出”！ ...
