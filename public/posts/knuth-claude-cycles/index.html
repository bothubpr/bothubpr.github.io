AI 辅助发现有向图哈密顿循环的通用构造

March 4, 2026 · Ziv
Table of Contents
  • 背景
  • Claude 的求解过程
  • 数学证明
  • 开放问题

计算机科学泰斗 Donald Knuth 抛出的开放问题被 Claude Opus 4.6 在一小时内解决。Knuth 随后给出了严格的数学证明，确认该构造对所有奇数 m 有效。

背景#

Knuth 在研究《计算机程序设计艺术》未来卷册中的有向哈密顿循环问题时，构造了如下有向图：

图中共有 m³ 个顶点，记为 ijk（0 ≤ i, j, k < m）。每个顶点有三条出边，分别指向：

  • i+jk：i 加 1（模 m）
  • ij+k：j 加 1（模 m）
  • ijk+：k 加 1（模 m）

开放问题：能否将这 3m³ 条弧分解为三个有向哈密顿循环？

Knuth 自身仅解决了 m = 3 的特殊情况。其友Filip Stappers 通过枚举发现了 m = 4 至 16 的数值解，但通用构造（对任意 m > 2）始终未知。

Claude 的求解过程#

Filip 将此问题提交给 Claude Opus 4.6，并要求其记录每次探索的进展。Claude 共进行了 31 轮探索，主要尝试包括：

早期尝试：线性函数、二次函数、深度优先搜索、2D 蛇形分析等方法，均未能找到通用构造。

关键突破（第15-20轮）：Claude 引入了纤维分解（Fiber Decomposition）方法，将顶点按 s = (i + j + k) mod m 分层分析。结合模拟退火算法，Claude 找到了 m = 3 和 m = 4 的具体解。

最终成功（第30-31轮）：Claude 注意到每个纤维的选择仅依赖于单个坐标：s = 0 时仅依赖 j，s = 1 和 s = 2 时仅依赖 i。由此构造出如下程序：

int c, i, j, k, m, s, t;
char *d;
for (c = 0; c < 3; c++) {
  for (t = i = j = k = 0; ; t++) {
    printf("%x%x%x ", i, j, k);
    if (t == m * m * m) break;
    s = (i + j + k) % m;
    if (s == 0) d = (j == m - 1 ? "012" : "210");
    else if (s == m - 1) d = (i > 0 ? "120" : "210");
    else d = (i == m - 1 ? "201" : "102");
    switch (d[c]) {
      case '0': i = (i + 1) % m; break;
      case '1': j = (j + 1) % m; break;
      case '2': k = (k + 1) % m; break;
    }
  }
}

Filip 对 m = 3, 5, 7, …, 101 共50个奇数进行了验证，均成功生成了有效的哈密顿循环分解。

数学证明#

程序虽有效，但 Knuth 需要严格的数学证明。他完成了对三个循环的完整证明，确认该构造对所有奇数 m > 1 有效。

进一步地，Knuth 证明：此类"Claude-like"构造共有 760 个可行解（总解空间为 4554 个）。

开放问题#

偶数 m（m = 4, 6, 8 等）的情况仍未解决。Claude 声称找到了这些值的数值解，但未能推广出通用构造。
