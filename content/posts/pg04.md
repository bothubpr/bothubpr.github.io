## ★ 痛点破局：信息过载与“机翻”的灾难

每天早晨刷手机，大伙儿肯定面临过这两个痛点：
1. **【信息源太分散】**：Hacker News、Reddit、Product Hunt，挨个刷一遍半小时就没了。外文长篇大论，根本没时间细看。
2. **【劣质翻译的灾难】**：如果用浏览器自带的翻译，它会把所有的专业术语全毁掉。比如把 Kubernetes 翻成“舵手”，把 Containerization 翻成“集装箱化”，读起来简直让人吐血。

咱们的 **【AI 赛博记者】** 就是来“降维打击”这些痛点的。


## ★ 核心逻辑与“反爬”突围战

这个脚本的核心逻辑很简单：利用 Python 的 `urllib` 抓取四大极客社区的 RSS 源，打包丢给大模型（如 DeepSeek 或 OpenAI）进行总结，最后生成 Markdown 扔进咱们的 `SyncDrive` 网盘。

但第一步“抓取”就差点翻车。当俺试图去抓 Reddit 的技术板块时，直接被它背后的 Cloudflare 防御盾弹了一个 `403 Forbidden`。

**【架构师的解法】**：咱们必须进行深度伪装。在原生 Python HTTP 请求中，强行注入极其逼真的浏览器 `User-Agent`，并声明我们接收 `application/rss+xml`。如果 Reddit 还是傲娇，俺还加入了 **【Lobsters】**（国外极硬核且对爬虫友好的开发者社区）作为最强替补！


## ★ 大模型调教：字斟句酌的 Prompt 工程

为了让 AI 吐出极具“信息穿透力”的新闻，俺对 Prompt 进行了极其严苛的约束。这里面有两个 **【灵魂设定】**：

1. **【拒绝废话，标题直译】**：绝不能让 AI 把“标题和内容”打包提炼，否则它会自作聪明地把标题改写成一句长句。必须规定：**【标题直接翻译，内容提炼总结 150 字】**。
2. **【强制保留英文术语】**：这招是精髓！必须在指令里加上：“所有的专业名词必须采用『中文 (English)』的格式，例如：容器化 (Containerization)”。只有这样，非英语母语的极客才能真正读懂核心技术的上下文。


## ★ 移动端交互的极致打磨：防误触链接

在生成 Markdown 排版时，俺遇到了一个 UX（用户体验）大坑。
如果直接把新闻原文链接生成普通的 `[点击阅读](URL)`，在手机或 iPad 上阅读时，长按复制非常容易 **【误触跳转】**，极其反人类。

**【架构师的解法】**：让 AI 把所有的原文链接，全部用反引号包裹成 **【纯文本代码块】**（如 `` `https://...` ``）。这样在手机上，代码块会带有一层浅灰色的底纹，你只需轻轻长按，就能无缝框选整段 URL，绝对不会发生误跳！


## ★ 终极点火：赛博记者核心源码公开

废话不多说，直接上核心源码！在你的 Ubuntu 脚本目录新建一个 `daily_news.py`，填入你的 API 密钥：

```python
import urllib.request
import xml.etree.ElementTree as ET
import json, datetime, os

# 【架构师配置区】
API_URL = "[https://api.deepseek.com/chat/completions](https://api.deepseek.com/chat/completions)" # 可换任意兼容 OpenAI 的接口
API_KEY = "YOUR_API_KEY_HERE"  # 填入你的真实 API KEY
MODEL_NAME = "deepseek-chat"
SAVE_DIR = "/home/ziv/Desktop/SyncDrive/" # 咱们的网盘绝对路径

SOURCES = {
    "Hacker News": "[https://hnrss.org/frontpage](https://hnrss.org/frontpage)",
    "Reddit Tech": "[https://www.reddit.com/r/technology/top.rss?t=day](https://www.reddit.com/r/technology/top.rss?t=day)",
    "Lobsters": "[https://lobste.rs/rss](https://lobste.rs/rss)",
    "Product Hunt": "[https://hnrss.org/newest?q=Product+Hunt](https://hnrss.org/newest?q=Product+Hunt)"
}

# 终极伪装，突破拦截
HEADERS = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
    'Accept': 'application/rss+xml, application/xml, text/xml'
}

def fetch_rss(name, url):
    try:
        req = urllib.request.Request(url, headers=HEADERS)
        with urllib.request.urlopen(req, timeout=15) as response:
            xml_data = response.read()
        root = ET.fromstring(xml_data)
        items = []
        for item in root.findall('.//item')[:5]:
            title = item.find('title').text if item.find('title') is not None else ''
            link = item.find('link').text if item.find('link') is not None else ''
            items.append(f"[{name}] {title}\nURL: {link}")
        return items
    except Exception as e:
        return [f"[{name}] Fetch Error: {str(e)}"]

def call_llm(raw_text):
    prompt = f"""
    你是一个资深的极客科技记者。请对以下抓取到的当日科技新闻进行筛选、翻译和总结。
    要求：
    1. 过滤掉明显的营销垃圾和无意义内容，挑出最具技术价值的 15 条以内的新闻。
    2. 标题直接翻译成中文。内容提炼总结（150字以内），翻译成通顺的简体中文。
    3. 极其重要：所有的专业名词必须保留英文原词或采用“中文 (English)”的格式，例如：容器化 (Containerization)。
    4. 严格按照以下 Markdown 格式输出每一条（严格使用三级标题）：
    
    ### [中文标题]
    - **来源标签**: [来源名称]
    - **内容精炼**: [150字以内的中文摘要总结]
    - **原文链接**: `[URL]`
    
    原始数据：
    {raw_text}
    """
    
    data = {"model": MODEL_NAME, "messages": [{"role": "user", "content": prompt}], "temperature": 0.5}
    req = urllib.request.Request(API_URL, data=json.dumps(data).encode('utf-8'), headers={
        'Content-Type': 'application/json', 'Authorization': f'Bearer {API_KEY}'
    })
    
    with urllib.request.urlopen(req, timeout=120) as response:
        return json.loads(response.read().decode('utf-8'))['choices'][0]['message']['content']

if __name__ == "__main__":
    today = datetime.datetime.now().strftime("%Y-%m-%d")
    all_news = [news for name, url in SOURCES.items() for news in fetch_rss(name, url)]
    ai_report = call_llm("\n\n".join(all_news))
    
    # 极简硬核的深度阅读指南
    footer = """
---
**首席架构师的深度阅读指南**

请帮我深度读取并解析这篇文章：[此处粘贴URL]。我希望你：
1. 提炼核心技术架构或核心观点；
2. 告诉我这项技术/产品解决了什么行业痛点；
3. 总结读者评论中最有价值的反面意见或潜在缺陷。
"""
    final_markdown = f"{ai_report}\n\n{footer}"
    file_path = os.path.join(SAVE_DIR, f"AI_Daily_News_{today}.md")
    with open(file_path, "w", encoding="utf-8") as f:
        f.write(final_markdown)
```
