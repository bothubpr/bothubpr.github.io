> 计算机科学泰斗 Donald Knuth（高德纳）研究数周的数学难题，被 AI 在一小时内破解。这是怎样的一天？

## 发生了什么？

2026年2月28日，83岁的**Donald Knuth**（高德纳）——计算机科学界传奇人物、《计算机程序设计艺术》(TAoCP) 作者——收到了一个让他"Shock! Shock!"的消息：

他研究了几个星期的数学难题，被 **Claude Opus 4.6** 解决了。

> "我不仅为我的猜想有了一个漂亮的解决方案而高兴，更为自动推理和创造性问题解决的这一戏剧性进步而庆祝。"  
> —— Donald Knuth

---

## 这道题究竟是什么？

Knuth 在研究《计算机程序设计艺术》未来卷册中的**有向哈密顿循环** (Directed Hamiltonian Cycles) 时，遇到了这个问题：

**构建一个 m³ 个顶点的有向图**，每个顶点记为 ijk（0 ≤ i, j, k < m），每个顶点有三条出边，分别指向：
- `i+jk`（i 加1）
- `ij+k`（j 加1）
- `ijk+`（k 加1）

**任务**：把这 3m³ 条弧（arcs）分解成三个有向哈密顿循环。

简单说：找到一个算法，把所有路径完美地走一遍，不重复不遗漏。

---

## 为什么难倒了一代人？

这个问题是 Knuth 给《计算机程序设计艺术》习题集的扩展问题。

- Knuth 自己解决了 m = 3 的情况
- 他的朋友 Filip Stappers 通过经验发现了 m = 4 到 16 的解
- 但**通用解**（任意 m > 2）一直未找到

Filip 决定把这个问题交给 **Claude**，并给予指导：
> "在每次 exploreXX.py 运行后，立即更新 plan.md 文件。在开始下一次探索之前，先记录进度。没有例外。"

---

## Claude 的 31 次探索

以下是 Claude 解决这个问题的历程：

| 探索次数 | 方法 | 结果 |
|---------|------|------|
| #1 | 简单线性/二次函数测试 | 失败 |
| #2 | 深度优先搜索 (DFS) | 搜索空间太大 |
| #3 | 2D 蛇形分析 | 发现 m=3 的解法 |
| #4 | 3D 蛇形模式 | 剩余图难以分解 |
| #15 | 纤维分解 (Fiber Decomposition) | 找到 m=3 的解 |
| #18-20 | 模拟退火 | 找到 m=3, 4 的解 |
| #27 | 坐标轮换 + 单超平面 | 几乎成功，但有冲突 |
| #29 | 证明很多方法不可能 | 死胡同 |
| #30-31 | **最终突破！** | 找到通用解 |

---

## 关键突破：纤维坐标

在第 15 次探索中，Claude 引入了**纤维分解** (Fiber Decomposition) 概念：

将顶点按照 `s = (i + j + k) mod m` 分层，每一层是一个"纤维" (fiber)。

在纤维坐标中，问题变成：为每个 (i, j, s) 选择一个排列，决定移动到哪个方向。

最终在第 31 次探索，Claude 发现：

> **每个纤维的选择只依赖于单个坐标**  
> - s = 0 时，只依赖 j  
> - s = 1 和 s = 2 时，只依赖 i

这导致了最终的程序：

```c
int c, i, j, k, m, s, t;
char *d;
for (c = 0; c < 3; c++) {
  for (t = i = j = k = 0; ; t++) {
    printf("%x%x%x ", i, j, k);
    if (t == m * m * m) break;
    s = (i + j + k) % m;
    if (s == 0) d = (j == m - 1 ? "012" : "210");
    else if (s == m - 1) d = (i > 0 ? "120" : "210");
    else d = (i == m - 1 ? "201" : "102");
    switch (d[c]) {
      case '0': i = (i + 1) % m; break;
      case '1': j = (j + 1) % m; break;
      case '2': k = (k + 1) % m; break;
    }
  }
  printf("\n");
}
```

Filip 测试了这个程序对所有 3 到 101 之间的**奇数 m**，全部成功！

---

## 数学证明

虽然程序工作了，但 Knuth 需要的不仅是"可行"，更是**数学证明**。

他证明了：

1. 第一个循环（c=0）确实是哈密顿的
2. 对于奇数 m，三个循环可以完美分解所有弧
3. 存在 **760 个"Claude-like"解**（解的数量是 4554 个中的约 1/6）

**未解决的问题**：偶数 m 的情况仍然未知。Claude 声称找到了 m=4, 6, 8 的解，但无法推广。
