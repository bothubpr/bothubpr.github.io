# 第一章：计算机到底在处理什么？

先从一个最根本的问题开始。

**计算机到底在处理什么？**

答案只有两个字：

**数字**

更准确说是：

**二进制数字（Binary Number）**

---

## 1. 什么是二进制（Binary）？

我们日常使用的是 **十进制（Decimal）**

例如：

```
1
10
100
999
```

但计算机使用的是 **二进制（Binary）**

只有两个数字：

```
0
1
```

例如

```
01000001
11010101
00010011
```

---

## 2. 为什么计算机使用二进制？

因为计算机硬件是 **电子电路（Electronic Circuit）**

电子电路只有两种稳定状态：

```
有电  → 1
没电  → 0
```

所以：

```
电压高 = 1
电压低 = 0
```

于是所有信息都变成：

```
010101010101
```

---

## 3. 图片、视频、文字本质是什么？

很多人会问：

图片怎么可能是数字？

其实是这样的：

### 文字

例如：

```
A
```

在计算机里是：

```
01000001
```

这叫

**字符编码（Character Encoding）**

例如：

**ASCII（American Standard Code for Information Interchange）**

---

### 图片

图片其实是：

**像素（Pixel）**

每个像素有颜色。

颜色又是：

**RGB（Red Green Blue）**

例如：

```
红 = 255
绿 = 0
蓝 = 0
```

于是：

```
(255,0,0)
```

再变成二进制。

---

### 视频

视频其实是：

```
很多张图片 + 时间
```

例如：

```
1秒 = 30张图片
```

所以视频本质是：

```
图片 + 声音 + 时间
```

全部最后都会变成：

```
010101010101
```

# 第二章：什么是“浮点数”（Floating Point）

接下来进入很多人听不懂的一个词：

**浮点数（Floating Point Number）**

---

## 1. 为什么需要浮点数？

计算机一开始只会处理：

```
整数（Integer）
```

例如：

```
1
2
3
100
```

但世界上有很多：

```
0.1
3.14
0.00001
```

这种叫：

**小数**

数学里叫：

**实数（Real Number）**

---

## 2. 计算机怎么表示小数？

如果直接存储：

```
0.00000000000000123
```

会非常浪费空间。

于是科学家发明了一种方法：

**科学计数法（Scientific Notation）**

例如：

```
123000000

= 1.23 × 10^8
```

---

计算机把这个思想改造后变成：

**浮点数（Floating Point）**

结构类似：

```
数字 × 2^指数
```

---

## 3. 为什么叫“浮点”？

因为小数点是可以移动的。

例如：

```
1.23
12.3
123
```

小数点在“浮动”。

所以叫：

```
Floating Point
```

---

# 第三章：什么是浮点精度（Floating Point Precision）

你经常会听到：

```
FP32
FP16
FP8
```

这里的：

**FP = Floating Point**

意思是：

**浮点数精度**

---

## 1. FP32 是什么？

**FP32（32-bit Floating Point）**

意思是：

```
一个数字占32位
```

例如：

```
00101010101010101010101010101010
```

越多位意味着：

```
精度越高
```

但也意味着：

```
占用显存越大
```

---

## 2. 常见精度

| 类型   | 英文                | 位数    | 特点   |
| ---- | ----------------- | ----- | ---- |
| FP32 | Float32           | 32bit | 最精确  |
| FP16 | Float16           | 16bit | AI常用 |
| INT8 | Integer8          | 8bit  | 推理常用 |
| Q4   | Quantization 4bit | 4bit  | 超省显存 |

---

# 第四章：什么是量化（Quantization）

这是 **本地部署 AI 最重要的技术**。

---

## 1. 什么是量化？

**量化（Quantization）**

就是：

```
把高精度数字压缩成低精度
```

例如：

原始：

```
0.123456789
```

压缩：

```
0.12
```

---

好处：

```
节省显存
加快速度
```

坏处：

```
精度下降
```

---

## 2. Q4 / Q5 / Q8 是什么？

量化等级：

| 类型 | 含义     |
| -- | ------ |
| Q8 | 8bit量化 |
| Q6 | 6bit量化 |
| Q5 | 5bit量化 |
| Q4 | 4bit量化 |

越低：

```
显存越小
速度越快
```

但：

```
智能稍微下降
```

```

训练就是：

```

```

例如：

```

```

```

训练需要：

```
几千张GPU
几个月时间
```

---

## 2. 推理

推理就是：

```
使用AI
```

例如：

```
问问题
写代码
聊天
```

**本地推理（Local Inference）**
