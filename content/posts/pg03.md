在前两篇连载中，咱们成功用 Filebrowser 搭起了网盘，并用虚拟挂载让 Hugo 引擎 **【死死盯住】** 了它。

这听起来很完美：你往网盘扔一个 `.md` 文件，网页就自动生成。
但现实很快给了俺一记响亮的耳光：Hugo 引擎是个极度死板的“八股文考官”。它要求你的每一篇文章，开头必须带有严格的 **【YAML 格式头（Front Matter）】**。比如这样：

```yaml
---
title: "我的日记"
date: 2026-03-04
draft: false
---
```

如果少了这个头，Hugo 要么把你当空气（不渲染），要么排版直接崩盘。
大伙儿想想，俺们好不容易实现了“随手记、随手拖拽”，难道每次扔文件前，还要用文本编辑器小心翼翼地敲上这几行代码？这绝对是 **【反人类的伪自动化】**！


## ★ 痛点破局：我们需要一个 24 小时待命的“赛博门卫”

既然要追求极致的自动化，那咱们就贯彻到底。俺决定在 Ubuntu 服务器里，安插一个 **【格式清洗门卫】**。

它的工作逻辑极度简单粗暴：
1. 24 小时死死盯着咱们的 `SyncDrive` 网盘。
2. 只要发现新进来的 `.md` 文件没有 `---` 这个标志性的 YAML 头部。
3. 它就瞬间冲上去，把文件的名字提取出来当做 `title`，抓取当前的系统时间当做 `date`，然后强行把标准的 YAML 头部 **【缝合】** 进文章里！

只要门卫处理得足够快，Hugo 引擎就会立刻接手，整个过程在毫秒间完成。你在这头刚把文件拖进网页，在那头刷新博客，一篇排版精美的文章就已经诞生了。


## ★ 极客的傲骨：抛弃第三方库，纯手搓轮询

在写这个脚本时，很多人第一反应是去 `pip install watchdog`（一个专门监控文件系统的 Python 库）。

但俺作为首席架构师，有着不可妥协的 **【代码洁癖】**：能用系统标准库解决的，绝不引入哪怕一个第三方依赖！这叫降低系统的信息熵，提升 **【绝对的稳定性】**。

俺直接动用 Python 自带的 `os` 和 `time` 库，写了一个极度轻量的死循环轮询脚本。

### 核心 Python 代码大公开

请在你的 Ubuntu `~/Desktop/auto_sub/` 目录下（俺专门用来放自动化脚本的文件夹），新建一个文件叫 `format_cleaner.py`，把下面这段代码贴进去：

```python
import os
import time
import datetime

# 【核心配置区】死死盯住咱们的网盘
WATCH_DIR = "/home/ziv/Desktop/SyncDrive"

def process_file(filepath):
    # 读取原始内容
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read()
    
    # 【核心逻辑】：如果文件不包含 YAML 头（即不是以 --- 开头）
    if not content.startswith("---"):
        print(f"[{datetime.datetime.now().strftime('%H:%M:%S')}] 发现毛坯文件，开始清洗: {filepath}")
        
        # 提取文件名（去掉 .md 后缀，把减号替换为空格当做优雅的标题）
        filename = os.path.basename(filepath)
        title = os.path.splitext(filename)[0].replace("-", " ")
        today = datetime.datetime.now().strftime("%Y-%m-%d")
        
        # 强行生成标准 YAML 头部
        front_matter = f"---\ntitle: '{title}'\ndate: {today}\ndraft: false\n---\n\n"
        
        # 缝合并覆盖保存
        with open(filepath, 'w', encoding='utf-8') as f:
            f.write(front_matter + content)

def start_watchdog():
    print(f"★ 赛博门卫已就绪，正在监听目录: {WATCH_DIR}")
    while True:
        try:
            # 遍历网盘里的所有 .md 文件
            for filename in os.listdir(WATCH_DIR):
                if filename.endswith(".md"):
                    filepath = os.path.join(WATCH_DIR, filename)
                    process_file(filepath)
        except Exception as e:
            print(f"监听出错，忽略异常继续运行: {e}")
        
        # 每次巡逻完，门卫打个盹（3秒），避免榨干 CPU
        time.sleep(3)

if __name__ == "__main__":
    start_watchdog()
```


## ★ 终极点火：让门卫隐入后台 (nohup 魔法)

脚本写好了，如果直接用 `python3 format_cleaner.py` 跑，终端一关，门卫就下班了。这显然不行。

我们需要用 Linux 的祖传魔法 **【nohup（不挂断）】**，把它剥离出当前的终端会话，让它像幽灵一样永远运行在操作系统的后台。

直接在终端敲下这行命令：

```bash
nohup python3 /home/ziv/Desktop/auto_sub/format_cleaner.py > /home/ziv/Desktop/auto_sub/cleaner.log 2>&1 &
```

敲完回车，哪怕你立刻拔掉连接 Ubuntu 的网线，这个特工也会在后台 24 小时不知疲倦地为你清洗数据。
